Mac SSH 連線

假如之前已經使用 https 對 Github push, 可能連線 github.com 的帳號與密碼已經存在 Go Untility Keychain Access 中, 要改為 SSH 對 Github push, 需要先刪除存入 Keychain Access 中的 github.com Certification 設定.

倉儲 .git 目錄中的 config:

從 https 連線改為 SSH 連線:

[remote "origin"]
#url = https://github.com/mdecourse/cp2019.git
url = git@github.com:mdecourse/cp2019.git

建立 OpenSSH 格式的 key pair:

ssh-keygen -t rsa -b 4096 -C "使用者學號"

將 id_rsa 存入內定的 ~/.ssh 目錄中.

透過 Proxy Serve r連線:

ssh USER@FINAL_DEST -o "ProxyCommand=nc -X connect -x PROXYHOST:PROXYPORT %h %p"

以  IPv6 連線到 github.com 為例:

.ssh/config 設定為:

ProxyCommand nc -X connect -x [2001:288:6004:your::proxy_server]:3128 %h %p

Host github.com
User git
Port 22
Hostname gitlab.com

其中 connect 表示使用 HTTPS_PROXY 連線, 內定則為 5 表示透過 SOCKS Version 5 連線.

https://ss64.com/osx/nc.html
